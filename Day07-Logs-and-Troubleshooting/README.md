# Day 07 - Logs and Troubleshooting

## Objective

Learn how to read logs and troubleshoot issues in Linux systems.

---

# What is a Log?

A log is a record of events generated by the operating system or applications.

Logs help identify errors and troubleshoot issues.

---

# Commands Learned

## cat

Purpose:
Displays complete file content.

Example:

```bash
cat file.txt
```

---

## less

Purpose:
View large files page by page.

Example:

```bash
less logfile.log
```

---

## head

Purpose:
Displays first 10 lines of a file.

Example:

```bash
head logfile.log
```

---

## tail

Purpose:
Displays last 10 lines of a file.

Example:

```bash
tail logfile.log
```

---

## tail -f

Purpose:
Monitor logs in real time.

Example:

```bash
tail -f application.log
```

---

## grep

Purpose:
Search words inside files.

Example:

```bash
grep ERROR logfile.log
```

---

## journalctl

Purpose:
View system logs.

Example:

```bash
journalctl
```

---

# Real DevOps Scenarios

## Website Down

```bash
systemctl status nginx

journalctl -u nginx
```

---

## Jenkins Not Working

```bash
systemctl status jenkins

journalctl -u jenkins
```

---

## User Login Failed

```bash
grep failed /var/log/auth.log
```
